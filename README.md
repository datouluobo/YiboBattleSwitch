# BattleSwitchLab

BattleSwitchLab 是一个面向 Battle.net / 魔兽世界多账号本地切换的独立工具实验项目。

最终定位：

- 它不是挂在 NewBeeBox 上的启动工具
- 它的最终目标是完全脱离 NewBeeBox，成为一个自主管理账号数据、切换状态与诊断能力的完整独立工具
- NewBeeBox 当前只作为研究样本和迁移来源，不是未来运行时依赖
- 启动 Battle.net 只是其中一个辅助动作，不是产品核心

产品边界：

- 研究阶段：读取 `NewBeeBox` 已有缓存，分析其账号切换链路与本地状态结构
- 迁移阶段：把登录数据复制到本程序自己的存储中，建立独立的数据模型与读写逻辑
- 独立阶段：不再依赖 `NewBeeBox.exe`、其页面流程或其缓存目录，完整提供账号保存、切换、备份、恢复、删除、诊断与导出能力

为什么不做“挂在新手盒子上的启动工具”：

- 这会让产品边界长期模糊，容易退化成“辅助启动器”
- 这会把后续兼容性绑在 NewBeeBox 的页面结构、版本更新和缓存格式上
- 这不利于建立本程序自己的账号资产、备份机制和异常恢复闭环

独立工具路线的优缺点：

- 优点：边界清晰；可建立本程序自有存储；可逐步摆脱第三方缓存依赖；便于实现完整功能闭环
- 缺点：前期逆向与兼容成本更高；需要自己承担数据迁移、异常处理和用户引导

当前目标：

- 读取 NewBeeBox 已保存的账号缓存，作为研究输入
- 研究并复刻账号切换所需的本地状态恢复链路
- 设计并落地本程序自有的账号存储结构
- 做成完全独立运行、可备份、可回退、可迁移的账号切换工具

当前已确认：

- NewBeeBox 的“账号切换”不是独立 EXE，而是主程序内部页面
- 账号缓存位于 `C:\Users\Administrator\AppData\Roaming\NewBeeBox\battleCache`
- Battle.net 本地启动状态与 `HKCU\Software\Blizzard Entertainment\Battle.net` 下的注册表值有关
- 已有一个可运行的原型，可将外部账号导入本程序账号库，并尝试写回 `WoW\WEB_TOKEN`

当前原型已具备：

- 从 `NewBeeBox` 导入账号到本程序自有账号库
- 从当前 Battle.net 登录态保存账号到本程序自有账号库
- 从本程序账号库读取账号并执行原型切换
- 备份、恢复、打开账号库目录
- 删除本程序账号库中的账号

下一步建议：

1. 记录 NewBeeBox 内部切换前后的注册表快照
2. 精确映射每个缓存 Blob 对应的目标键
3. 设计本程序自己的账号存储格式与目录结构
4. 实现“从 NewBeeBox 导入到本程序存储”
5. 完整实现“保存当前账号 / 切换 / 删除 / 恢复备份 / 导出”
