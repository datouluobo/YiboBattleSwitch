## 设计意图

- 顶部先回答三个问题：
  - 现在是谁
  - 程序当前是否具备执行条件
  - 能不能安全回退
- 左侧只放账号库和账号管理
- 右侧只放“对这个账号能做什么”
- 主按钮只有一个：`切换到此账号`
- 诊断、目录、管理员重启等工具全部降级到“高级工具”

## 建议删掉或降级的旧元素

- `要写回的 Blob`
- `写入目标键`
- `可选 GAME_ACCOUNT`
- `实验写回（已暂停）`

这些内容仍然可以保留在代码里做诊断，但不应该继续出现在主界面主路径中。

## 新主流程

1. 用户选择账号
2. 点击 `切换到此账号`
3. 程序自动执行：
   - 先备份当前健康状态
   - 关闭 Battle.net / Agent
   - 恢复目标账号配置
   - 恢复目标账号认证
   - 启动 Battle.net
4. 若失败：
   - 显示明确失败信息
   - 提示一键恢复最近一次健康备份

## 文案要求

- `保存账号名` 改成 `WoW 子账号`
- `邮箱` 改成 `账号标识`
- `Battle.net 登录名候选` 保留，但作为说明项，不作为主操作概念
- 任何地方都不要让用户先理解 `Blob` 才能用工具
