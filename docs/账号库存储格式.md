# BattleSwitchLab 账号库存储格式

当前原型的本程序账号库存储目录：

- `prototype\battle_switch_library\accounts\<account-id>\`

每个账号目录当前包含 5 个文件：

1. `info.json`
2. `account.json`
3. `registry.json`
4. `battlenet_config.txt`
5. `battlenet_config.json`

## 结构说明

`info.json`：

- `account`: 账号名或邮箱
- `description`: 备注
- `backupTime`: 毫秒时间戳
- `lastLoginTime`: 毫秒时间戳
- `importedFrom`: 来源，例如 `NewBeeBox` 或 `CurrentBattleNet`
- `importedAt`: 导入或保存时间

`account.json`：

- `savedAccountName`: 当前已知的游戏子账号名，例如 `YOUTH0923`

`registry.json`：

- 当前保存的是 Battle.net `UnifiedAuth` 的二进制值快照
- 根键 `""` 下保存多个 Blob
- 每个 Blob 结构为：

```json
{
  "": {
    "20589EE4": {
      "Value": "base64..."
    }
  }
}
```

`battlenet_config.txt`：

- 保存账号采集时的原始 `Battle.net.config` 文本
- 作为最接近 Battle.net 本地文件的回放素材

`battlenet_config.json`：

- 当 `Battle.net.config` 能被解析为 JSON 时，额外保存一份格式化 JSON
- 便于后续直接读取 `Client.SavedAccountNames` 等字段

## 当前状态

- 从 `NewBeeBox` 导入时，会把其缓存中的 Blob 复制到本程序账号库
- 从 `NewBeeBox` 导入时，暂时还没有对应账号的 `Battle.net.config` 快照，所以配置文件字段会留空
- “保存当前登录为账号”时，会把当前 `UnifiedAuth` 二进制值快照和当前 `Battle.net.config` 一起保存到本程序账号库
- “备份当前状态”时，也会把当前 `Battle.net.config` 一起写进备份快照，便于完整回退
- 原型切换时，优先从本程序账号库读取 Blob

## 后续演进方向

- 为不同目标注册表键建立明确字段，而不是只保留研究期 Blob
- 把 `Battle.net.config` 的恢复能力接到账号切换链路中
- 增加账号级元数据，例如显示名、标签、排序、最近使用时间
- 增加导出格式与导入校验
- 在完全摸清切换链路后，逐步从“Blob 存储”过渡到“语义化状态存储”
